
# Open Source Agriculture > Iot-ai-adopt-adapt

Open Source Ag 501(c)(3) is a grow-to-donate farm, nonprofit, and IoT Lab that researches affordable, open-source tools for small and mid-sized farms.This branch focuses on IoT and AI projects that can be adopted, adapted, updated, improved, and stewarded so they become more useful in real farm settings.

## What this branch is for

Many open-source agriculture projects are technically promising but remain underused because they were built on one or two working farms, developed as student or hobby projects, or left unfinished when the original maintainer got busy. This branch is a place to research those projects, evaluate their usefulness, and help move strong ideas from prototype or niche tool into practical use for more farms, vineyards, ranches, orchards, and community growing operations.

The goal is not to duplicate tools that already exist. The goal is to find valuable open-source IoT and AI work, understand what problem it solves, determine whether it fits small and mid-sized farms, and help close the gap between a promising repository and a field-ready tool.

## What happens here

Work in this branch may include:

- Researching open-source IoT and AI repositories relevant to agriculture.
- Sorting projects by category and subcategory, such as water management, soil management, climate monitoring, pest and disease, computer vision, or regenerative practice tracking.
- Reviewing repository activity, documentation, usability, openness, and farm relevance.
- Testing whether a project can be run with affordable hardware such as Raspberry Pi, ESP32, older laptops, or phones.
- Adapting projects toward local-first use so farm data can remain on-site rather than depending on cloud platforms.
- Improving documentation, setup instructions, and human-computer interaction so growers and apprentices can actually use the tools.
- Identifying farms willing to help test, adopt, and steward promising projects over time.

## Scope

This branch is especially interested in projects that:

- Add practical value to small and mid-sized farms.
- Can be implemented affordably or with reused hardware.
- Use open-source code or open hardware, or are open enough to be meaningfully adapted.
- Support local or edge data storage and simple on-farm workflows.
- Address real operating needs in farms, ranches, orchards, vineyards, packing areas, greenhouses, gardens, or food-access production systems.
- Show signs of being useful but undermaintained, underdocumented, niche, unfinished, or in need of stewardship.

## How projects are evaluated

Projects are reviewed with a practical rubric so they can be compared and sorted consistently. Common review criteria include:

- Latest commit or visible activity.
- Repository depth.
- Documentation quality.
- HCI or usability or accessibility.
- Relevancy to small farms.
- Relevancy to mid-sized farms.
- Level of open source versus proprietary dependency.
- Notes on implementation risks, adaptation needs, and stewardship potential.

This helps separate projects that are only interesting from projects that are realistic candidates for field testing, adoption, and long-term value.

## Preferred technical direction

Open Source Agriculture is especially interested in low-cost, local-first systems using Raspberry Pi, Python, LoRaWAN, Wi-Sun or similar radios, Bluetooth workflows, reused phones, and older laptops where practical. Projects that depend entirely on paid cloud services may still be worth reviewing, but they are stronger candidates when they can be converted to local or self-hosted use.
## Stewardship model

This branch supports a stewardship approach, not just a code archive. Some projects may remain research references. Others may become workshop examples, field-test candidates, documentation projects, co-maintained tools, or adoption pilots on working farms.

Possible stewardship paths include:

- Curated for learning.
- Evaluated with the rubric.
- Field-tested on a partner farm.
- Adapted for local-first operation.
- Improved with better setup guides or user interface changes.
- Shared with growers, apprentices, and collaborators.
- Maintained or archived with clear status notes.

## Collaboration

Useful collaborators may include growers, ranchers, orchardists, vineyard operators, makers, maintainers, students, apprentices, engineers, and researchers who want to help bring open tools into real agricultural use. This work benefits from practical field knowledge as much as technical skill, especially when deciding whether a tool solves a real problem under real farm conditions.

Contributions may include:

- Suggesting repositories to review.
- Testing hardware or software.
- Improving documentation.
- Simplifying deployment.
- Translating tools into clearer farm workflows.
- Connecting promising tools with implementation farms.
- Helping maintain or steward projects whose original builders moved on.

## Why this matters

Many small and mid-sized farms can benefit from open-source technology, but the gap between “interesting repository” and “working farm tool” is still wide. A project that worked for one farm or one developer can sometimes become a game changer for many more farms when it is documented clearly, simplified, adapted to affordable hardware, and stewarded with real users in mind.

## Contact and next steps

This branch is part of the broader Open Source Agriculture effort to grow food, train apprentices, and expand access to practical open-source agricultural tools.Over time, this work may connect with workshop materials, field guides, evaluation rubrics, implementation pilots, and farm-based collaboration across more working landscapes.
